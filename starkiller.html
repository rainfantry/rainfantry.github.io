Section 01

Operation Overview

StarKiller is an Android RAT built from first principles as part of the CSEC Tactical Cyber Operations curriculum. No frameworks. No copy-paste. Every component — C2 server, agent protocol, Kotlin client, obfuscation pipeline, APK binder — was researched, designed, and implemented from scratch against real Android security architecture.

The project extends the VADER campaign into the mobile domain. The same methodology applies: understand the defensive architecture, map the seams, build tools that exploit those seams, document everything. VADER owns the Windows desktop. StarKiller owns the pocket.

Architecture

Two-tier architecture. Python C2 server handles operator interaction via web dashboard and maintains persistent TCP connections to deployed agents. Kotlin client runs as a foreground service on Android, connecting back to the C2 listener with automatic reconnection and task dispatch.

C2 Server
Language Python Web Port 8666 — operator dashboard Agent Port 8667 — TCP listener for agents Protocol VADER protocol (length-prefixed JSON) Features Multi-agent management, task dispatch, file transfer, live monitoring
Android Client
Language Kotlin Min API 26 (Android 8.0 Oreo) Modules 12 capability modules Service Foreground service with START_STICKY Connection TCP with auto-reconnect, exponential backoff

Section 03

Client Modules

12 capability modules covering full device access. Each module is independently invokable from the C2 dashboard. Modules use Android's standard APIs — Camera2, AccessibilityService, ContentResolver, LocationManager — with runtime permission handling.

Camera

Camera2 API. Front and rear capture. Silent shutter. Returns JPEG to C2.

SMS Read / Send

ContentResolver query on content://sms. Read inbox/sent/drafts. Send SMS to arbitrary numbers via SmsManager.

GPS Tracking

LocationManager with GPS and network providers. Continuous tracking or single fix. Returns lat/long/accuracy/altitude.

Contacts Dump

ContentResolver on ContactsContract. Full contact list: name, phone, email, organisation. JSON export.

File Manager

Directory listing with metadata. File download/upload. Navigate internal and external storage.

Microphone Recorder

MediaRecorder or AudioRecord API. Configurable duration. Returns audio file to C2 via chunked transfer.

Keylogger

AccessibilityService. Captures all keystrokes system-wide via onAccessibilityEvent. Includes target app package name for context.

Call Log

ContentResolver on CallLog.Calls. Incoming, outgoing, missed. Duration, timestamp, contact name resolution.

App List

PackageManager enumeration. Installed apps with package name, version, install date, permissions. Detects security software.

Notification Listener

NotificationListenerService. Captures all notifications system-wide. App name, title, text, timestamp. Real-time forwarding.

Shell Exec

Runtime.exec(). Execute arbitrary shell commands on device. Returns stdout/stderr. Non-root context.

Full Recon

Composite module. Device info, network config, battery, display, sensors, installed apps, running processes. Complete target profile in one call.

Section 05

Persistence Mechanisms

Four independent persistence mechanisms ensure the agent survives device reboots, app kills, and battery optimisation. Each layer operates independently — the agent only needs one to survive.

Mechanism Trigger Android API Survivability
BOOT_COMPLETED Receiver Device reboot BroadcastReceiver Survives full power cycle
START_STICKY Service Service killed by OS Service.onStartCommand() OS automatically restarts service
WakeLock Device sleep PowerManager.WakeLock Keeps CPU running during standby
Icon Hiding User interaction PackageManager.setComponentEnabledSetting() Removes app from launcher — invisible to user

Foreground Service

// Android requires foreground notification for persistent services (API 26+) // StarKiller uses a minimal, generic notification to avoid suspicion startForeground(NOTIFICATION_ID, notification) .setSmallIcon(R.drawable.ic_system) // Generic system icon .setContentTitle("System Service") // Innocuous title .setPriority(NotificationCompat.PRIORITY_MIN) // Minimised visibility .setOngoing(true) // Cannot be swiped away // Combined with icon hiding = invisible app with persistent service
Section 06

Obfuscation Pipeline

Three-layer obfuscation. String encryption at the source level, ProGuard for bytecode transformation, and R8 for final binary stripping. The result: detection scanners dropped from 37 signatures to 4, and the remaining 4 are stripped by R8 at compile time.

String Encryption

All sensitive strings — C2 addresses, API names, package names, permission strings — are XOR-encoded and Base64-wrapped at build time. Runtime decryption via a static helper method. Strings never appear in the APK's string table in cleartext.

// Build time: "content://sms" → XOR(0x5A) → Base64 private const val SMS_URI = "Gx0NEB0cGQkzJRkR" // Runtime: decode → XOR → cleartext val uri = StringDecryptor.decrypt(SMS_URI) // → "content://sms"
ProGuard Configuration

5-pass ProGuard optimisation with aggressive settings. Class repackaging moves all classes to a flat namespace. Log stripping removes all logging calls from the final binary.

-optimizationpasses 5 -repackageclasses '' # Flat namespace -allowaccessmodification -overloadaggressively -assumenosideeffects class # Strip all Log calls android.util.Log { public static *** d(...); public static *** v(...); public static *** i(...); public static *** w(...); public static *** e(...); }

Detection Results

Stage Detections Method

After XOR+Base64 12 signatures String encryption removes static matches
After ProGuard 4 signatures Class repackaging, method renaming
After R8 final strip 0 remaining Dead code elimination removes flagged patterns
Section 07

APK Binder

Inject the StarKiller agent into legitimate Google Play APKs. The target app looks and functions normally. The payload runs silently in the background. Social engineering delivery — victim installs what appears to be a real app.

Injection Chain

  1. Decompile target APK with apktool — extract Smali bytecode and resources
  2. Inject StarKiller Smali classes into the decompiled source tree
  3. Patch AndroidManifest.xml — add permissions, services, receivers, and accessibility service declarations
  4. Inject launch hook into the target app's main Activity — onCreate() triggers StarKiller service start
  5. Recompile with apktool and sign with a debug or custom keystore
  6. Resulting APK: original app functionality + StarKiller agent running on boot

Manifest Patches

Section 09

Prior Art & Research Sources

StarKiller is original work built from first principles. The following published RATs were studied for architectural patterns, capability mapping, and evasion techniques. No code was reused. Understanding existing tools is part of the research methodology — you study what exists before you build what's next.

G700 RAT

Commercial-grade Android RAT. Studied for module architecture, C2 protocol design, and persistence mechanisms. Multi-panel dashboard pattern influenced StarKiller's web UI.

EagleSpy v5

Accessibility Service abuse patterns. Keylogger implementation via onAccessibilityEvent. Overlay attack techniques for credential phishing. Notification listener architecture.

L3MON

Open-source Android RAT. Studied for Node.js C2 server architecture, APK builder patterns, and real-time communication over WebSocket. Simpler architecture — useful as a baseline.

Cerberus (Leaked Source)

Banking trojan with overlay attacks. Studied for SYSTEM_ALERT_WINDOW abuse, SMS interception for 2FA capture, and anti-analysis techniques (emulator detection, debugger detection).
