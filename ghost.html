GHOST ENCODER // UNICODE STEGANOGRAPHIC RESEARCH

22DIV / george wu // CSEC research on own hardware // June 2026

OVERVIEW

Ghost Encoder researches covert data channels using Unicode zero-width characters. Binary data is encoded as invisible Unicode sequences — zero-width space, zero-width non-joiner, zero-width joiner — and embedded inside ordinary visible text. The resulting string is visually identical to the original. Copy-pasting it into a chat app, email, or document carries the hidden payload silently.

Research purpose: understand how data can be exfiltrated through channels that strip metadata (logs, chat history, email bodies) without triggering content inspection.

Any binary payload is serialised to bits, each bit is mapped to a zero-width character, and the sequence is interleaved into a carrier text string between visible characters. The carrier text reads normally. The hidden data survives copy-paste, plain text transmission, and most logging systems.

DETECTION IMPLICATIONS

DETECTION GAP
Content inspection misses zero-width exfiltration

DLP systems and content filters operate on visible text. Zero-width characters are typically treated as formatting — not flagged as potential exfiltration channels. A researcher sending encoded data via Slack, email, or Teams looks identical to sending plain text.

DEFENSIVE RECOMMENDATION
Strip or flag zero-width characters at DLP boundaries

Organisations concerned about insider exfiltration should normalise text at DLP inspection points. Detect anomalous zero-width character density as an indicator of steganographic encoding.
