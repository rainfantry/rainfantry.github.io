LESSON 01 / 04

THE GHOST ALPHABET

Unicode includes characters with zero visible width — they render as nothing in every text editor and terminal. We use 16 of them as a hex encoding alphabet.

// THE 16 INVISIBLE CHARACTERS

Unicode has characters that legitimately have no visual representation — Zero Width Space, Zero Width Joiner, Mongolian Vowel Separator. They're in the standard for real linguistic reasons.

We take 16 of them and assign hex values 0x0 through 0xF. Any byte (0x00–0xFF) can then be encoded as exactly 2 invisible characters — high nibble then low nibble.

A 10KB payload becomes 20KB of invisible Unicode. cat ghost.ps1 outputs a blank line. Get-Content ghost.ps1 shows nothing. The payload is RIGHT THERE — just invisible.

LESSON 02 / 04

ENCODE BYTES → INVISIBLE UNICODE

Each byte becomes two invisible chars. High nibble first, low nibble second. A 3-byte string like "ABC" becomes 6 invisible characters — completely invisible in any editor.

// NIBBLE SPLITTING

A byte is 8 bits. Split it into two 4-bit nibbles: high = top 4 bits, low = bottom 4 bits.

For byte 0x41 ('A'): high = (0x41 >> 4) & 0x0F = 0x4, low = 0x41 & 0x0F = 0x1. Two nibbles, each maps to one invisible char.

The decoder reverses this: sees two chars, looks up their indices (high, low), reconstructs (high << 4) | low = 0x41.

MISSION: Write encode_bytes()

// TARGET CODE:
def encode_bytes(data: bytes) -> str:
    encoded = []
    for byte in data:
        high = (byte >> 4) & 0x0F
        low = byte & 0x0F
        encoded.append(GHOST_ALPHABET[high])
        encoded.append(GHOST_ALPHABET[low])
    return ''.join(encoded)
// YOUR CODE:

// WHAT JUST HAPPENED

byte >> 4 — right shift 4 positions. Moves the high 4 bits down to positions 0-3. & 0x0F masks to keep only those 4 bits (clears any garbage in upper bits after shift).

byte & 0x0F — mask the bottom 4 bits directly. Upper bits become zero.

Result: two indices 0-15, each mapped to an invisible Unicode char. ''.join(encoded) concatenates them all. Output is a pure-invisible string.

LESSON 03 / 04

DECODE BACK TO BYTES

The reverse operation. Filter the invisible chars from the text, look up their indices in GHOST_REVERSE, pair them up, reconstruct bytes.

// DECODING STRATEGY

The encoded text may be embedded in a larger string (with visible chars around it). Filter to only the chars present in GHOST_REVERSE. They appear in pairs — char[0]+char[1] = byte[0], char[2]+char[3] = byte[1], etc.

For each pair: look up both indices, reconstruct byte as (high << 4) | low. Collect into a bytearray. Return as bytes.

MISSION: Write decode_ghost()

// TARGET CODE:
def decode_ghost(ghost_text: str) -> bytes:
    result = []
    chars = [c for c in ghost_text if c in GHOST_REVERSE]
    for i in range(0, len(chars), 2):
        high = GHOST_REVERSE[chars[i]]
        low = GHOST_REVERSE[chars[i + 1]]
        result.append((high << 4) | low)
    return bytes(result)
// YOUR CODE:

// WHAT JUST HAPPENED

[c for c in ghost_text if c in GHOST_REVERSE] — filter comprehension. Only keeps chars that are in our alphabet. This handles mixed-content strings where invisible chars are embedded in visible text (like the PS1 bootstrap stub).

range(0, len(chars), 2) — step 2, process pairs. chars[i] = high nibble char, chars[i+1] = low nibble char.

(high << 4) | low — shift high nibble back up to bits 4-7, OR in the low nibble at bits 0-3. Reconstructs the original byte.

Round-trip test: assert decode_ghost(encode_bytes(b"hello")) == b"hello" — this passes. The encoding is lossless.

LESSON 04 / 04

PS1 BOOTSTRAP — EXECUTE IN MEMORY

The final piece. A visible PS1 stub that decodes the invisible payload and executes it via Invoke-Expression — all in memory, nothing written to disk as cleartext.

// THE BOOTSTRAP TECHNIQUE

The ghost.ps1 file has two parts: a visible decoder stub at the top, and the invisible encoded payload embedded after it.

The stub reads the current file ($PSCommandPath), extracts the invisible chars, decodes them back to the original PS1 code, and calls Invoke-Expression on the result.

AV scans the visible stub — it's clean (just string operations). The invisible payload isn't a signature match. The real code executes in-memory and exits.
